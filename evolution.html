    L'histoire du CSS

    Pour permettre aux auteurs de façonner librement l'apparence de pages web, à partir de 1993 Netscape Navigator et Internet Explorer, le logiciel de navigation deux qui a contesté les utilisateurs dans la note Wars Browser, ils ont présenté tags propriétaires, ou ne pas adhérer à standard ni compatible avec les navigateurs concurrents. Un exemple de ces balises est qui définira les fonte caractère.

    Ces balises de formatage propriétaires étaient la seule façon pour les auteurs de définir la mise en forme, de sorte que leur utilisation est devenue très intense. Cependant, ces balises ont présenté trois problèmes:

    Le premier problème est constitué par la longueur de l'étiquette. Par rapport à une page qui adopte le langage CSS, une page qui ne prend pas en nature plus lourd (en termes de bit) Dans une relation qui arrive souvent 0,2%. De plus, les instructions CSS peuvent être collectées dans un fichier à l'extérieur qu'il reste stocké dans cachette navigateur, ce qui réduit davantage la quantité de données qui serveur Ils doivent transmettre.

    Le deuxième problème est le manque de logique du code HTML (X). Un code est conforme aux standards, redondants et fait confusion implique beaucoup de travail supplémentaire pour les navigateurs, ils devraient essayer de corriger et d'interpréter (si possible) des directives arbitraires.

    Le troisième problème commence à devenir de plus en plus important et il est le manque de compatibilité avec la nouvelle ordinateurs de poche et smartphone. Ces pages sont en fait conçus pour les écrans avec une résolution minimum 800x600 pixels. Les dispositifs tels que les PDA (ou les plus modernes comprimé) Qui peut avoir une résolution plus faible et une forme d'écran tout à fait différent du rapport 4: 3 pour moniteur ordinateur, sont donc incapables de voir correctement la page avec le résultat que l'utilisateur trouvera la page un confus et mal à l'aise à lire.

    Une autre question a été, dans les pages web non standard, l'utilisation de l'étiquette (la tables) Pour la mise en page des pages Web. La disposition de table est considérée comme déplorable par les puristes, car les tables sont conçues pour formater les données sous forme de tableau et non disposition web. En plus de cela, l'utilisation des tableaux comporte aussi des inconvénients pratiques: gain de poids des pages, comme déjà indiqué au problème 1, et l'accessibilité aggravation, que l'ordre logique des données peut être sacrifiée pour des raisons de mise en page. Pourtant, certains soutiennent l'utilisation des tables pour leur facilité de mise en œuvre.

    CSS 1

    Pour tâtons pour résoudre cette situation, en 1996 W3C publié CSS spécifique 1. CSS 1 était un moyen intéressant de séparer le contenu de mise en forme. La base de cette langue, en fait, est que le contenu serait toujours défini par le code HTML (X), alors que la mise en forme serait transféré à un code complètement séparé, CSS avec précision. Les appels entre les deux codes ont été faites par deux caractéristiques particulières: classe et ID. Ces spécifications:

    Ils étaient une solution efficace au premier problème (à l'exception de la question de l'étiquette) Parce qu'il réduit considérablement la taille de la page. Ils ont résolu la deuxième partie parce qu'il a permis le code HTML (X) pour revenir en grande partie simple et basique, mais a montré un manque qui a forcé de recourir à la balise HTML. Ils ne sont cependant pas, compte tenu du troisième problème, étant donné que, en 1996, PDA (PDA) ont été mal diffusés. CSS 1 a développé un simple mais efficace, mais en dépit de leur grand potentiel ont échoué en raison du manque de navigateur qui peut les soutenir.

    CSS 2 et CSS 2.1

    Pour inclure de nouvelles fonctionnalités et CSS faire une langue bien pris en charge, en 1998, le W3C a publié la spécification CSS 2. En 2004 a commencé le travail sur les dernières spécifications CSS 2.1 qui sont devenues recommandation officielle 7 Juin, 2011. Le CSS 2 sont l'évolution naturelle CSS 1 et offrent des solutions puissantes pour résoudre le troisième problème surtout, avec la possibilité de créer des feuilles de style séparées pour les appareils mobiles. La deuxième question est maintenant complètement résolu, écrire une page (X) HTML adressé exclusivement à la structure et gestion de contenu, puis uniquement avec CSS à impaginarla.

    A partir de Internet Explorer 5, Firefox et Opera 7, CSS 2 pourrait utiliser le navigateur peut les interpréter et faire partie du Code de nombreux sites Web.

    CSS 3

    Contrairement au CSS spécifique 2, qui est constitué par une seule spécification, les spécifications CSS3 sont composés de sections séparées appelées « modules ». En raison de cette modularisation, les spécifications CSS3 ont des états de progrès et de stabilité.[1] En Novembre 2014, cinq modules sont officiellement publiés par le W3C comment recommandations:

    2011-06-07: Couleur CSS Module Level 3 2011-09-29: CSS niveau 3 Module Namespaces 2011-09-29: Niveau 3 Selectors 2012-06-19: médias Requêtes 07/11/2013: Attributs de style CSS

    CSS3 devrait présenter des solutions aux la correction de certains bogues L'interprétation d'Internet Explorer[citation nécessaire], l'amélioration de la gestion des fonds et une solution pour atteindre bords arrondis dont la réalisation afflige un peu de temps webdesigner.

    HTML et CSS3 sont une langue turation complète[2][3], condition suffisante pour une langue soit considérée comme une langage de programmation.
